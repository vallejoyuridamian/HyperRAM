HyperRAM interface
Introduction
The HyperRAM interface will be used in a data acquisition system, and will consist of a VHDL module to interface with 4 RAM ICs that will share control lines, as in the diagram below. The idea is to have them 4 RAM ICs act as a single 32bit wide memory.
Glossary
user: In this document, we use the term “user” to describe the other VHDL code that will be using this interface.
Module: We use the term “module” to describe the HyperRAM VHDL interface.
word: 32 bits of data, distributed amongst 4 hyperRAM ICs
Ports definition
It’s important to follow the definition of the Input/Output ports described below:
-- Design for W958D8NBYA5I -- 4 chips will connected with control signal in parallel to result in a 32 bit bus -- Careful: command and address will be duplicate for all rams -- data read/write will be different for each ram -- the term "user" in the comments indicates the VHDL code interfacing to that entity -- All read and write transactions are 128 words linear bursts (not wrapped bursts) -- a burst consists of 128 * 4 RAMs = 512 bytes per transaction (128 bytes on each RAM) ENTITY hram_interface IS PORT ( --interface to user clk_system : IN STD_LOGIC; --125MHz reset : IN STD_LOGIC; ready : OUT STD_LOGIC; --indicates the interface is ready for a new write or read burst start_address : IN STD_LOGIC_VECTOR(31 DOWNTO 0); --First address in a burst write_start : IN STD_LOGIC; --user request a write burst read_start : IN STD_LOGIC; --user request a write burst --First-Word-Fall-Through FIFO interface. -- This fifo Will contain at least 128 words before write burst is requested -- It's the user responsibility to ensure `write_fifo_empty` is never asserted during a transaction write_fifo_data : IN STD_LOGIC_VECTOR(31 DOWNTO 0); write_fifo_data_rd_enable : OUT STD_LOGIC; write_fifo_empty : IN STD_LOGIC; --Fifo should never be empty during a transaction --Read fifo interface. When a burst read operation is performed, data is written in this fifo --It's the user responsibilty to ensure this fifo has at least 128 free slots before read burst is requested read_fifo_data : OUT STD_LOGIC_VECTOR(31 DOWNTO 0); read_fifo_wr_enable : OUT STD_LOGIC; read_fifo_full : IN STD_LOGIC; --Fifo should never become full --Hyper RAMs interface CS_n : OUT STD_LOGIC; CK_LVDS : OUT STD_LOGIC; --Lattice will infer the P/N LVDS pair automatically DQ : INOUT STD_LOGIC_VECTOR(31 DOWNTO 0); RWDS : INOUT STD_LOGIC; RESET_n : OUT STD_LOGIC ); END hram_interface;
Copy
VHDL
As described in the comments above all transaction to the hyperRAM IC will always be in linear bursts of 128 words. No any other mode should be implemented (to keep the code as simple and as maintainable as possible).
Mode Of Operation
Clocking
The module is clock by the main system clock, a 125MHz clock. The clock to the actual HyperRAM IC should be 62.5MHz (i.e. 125/2) and should be generated using the 125MHz system clock.
Reset
When RESET_n is asserted, the “ready” flag should be cleared (i.e. = ‘0’)
When RESET_n is released, the module should first:
Read ID and ensure it’s valid
Configure the hyperRAM chip as required for proper operation. This includes (but not limited to):
Set burst len to 128 Bytes
Set Fixed latency
Set minimum clock latency of 3
Set 27 Ohm drive strength
Set the “ready” output signal to ‘1’ if and only if the ID of the hyperRAM is correct.
Write request
A write operation happens in the following order, always:
The user waits until ready = ‘1’
User provides an address on the input start_address and sets write_start = ‘1’ for 1 clock cycle.
The module sets ready = ‘0’ to signify that it’s busy with the current operation
The module should fetch and write 128 words using the FIFO interface on the lines write_fifo_data , write_fifo_data_rd_enable , and write_fifo_empty . Please note: This is a FWFT (First Word Fall Through) style fifo, not a standrad fifo.
The data should be written starting at start_address, and increasing by 1 for each new word.
The module should carry on with the burst of 128 words until all 128 words are written. The fifo will never be empty - it’s the user responsibility to ensure there is always valid data in it.
When done, the module should set ready = ‘1’ and be ready to accept new requests
Read request
A write operation happens in the following order, always:
The user waits until ready = ‘1’
User provides an address on the input start_address and sets read_start = ‘1’ for 1 clock cycle.
The module sets ready = ‘0’ to signify that it’s busy with the current operation
The module should read 128 words from the hyperRAM and push them into the FIFO interface on the lines read_fifo_data , read_fifo_wr_enable , and read_fifo_full .
The data should be read starting at start_address
The module should carry on with the burst of 128 words until all 128 words are read and pushed to the FIFO.
When done, the module should set ready = ‘1’ and be ready to accept new requests
